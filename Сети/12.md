# 12. Обратная зона DNS. Пример.

Ахмедов: 12

Обратная [DNS-зона](https://doc.a-real.ru/index.php?article=24#dns-zone) — это специальная доменная зона. Она предназначена для определения имени хоста по его [IP-адресу](https://doc.a-real.ru/index.php?article=24#ip-address) с помощью [PTR-записи](https://doc.a-real.ru/index.php?article=229#ptr). 
PTR  запись по сути это просто обратная запись IP адреса хоста+in-addr.arpa . 
Адрес хоста AAA.BBB.CCC.DDD транслируется в обратной нотации и превращается в DDD.CCC.BBB.AAA.in-addr.arpa. Благодаря иерархической модели управления именами появляется возможность делегировать управление зоной владельцу диапазона IP-адресов. Для этого в записях авторитативного [DNS](https://doc.a-real.ru/index.php?article=24#dns)-сервера указывают, что за зону CCC.BBB.AAA.in-addr.arpa (то есть за сеть AAA.BBB.CCC.DDD/24) отвечает отдельный сервер.
Пример DNS приходит запрос с доменным именем mail.example.com  и   IP адресом 192.168.0.1.  Далее мы хотим проверить является ли этот адрес реально mail.example.com. Мы берем IP адрес и преобразуем его с помощью PTR записи в 1.0.168.192.in-addr-arpa и формируем запрос, на что нам возвращается имя хоста и сравнивается с тем что у нас было указано изначально.
