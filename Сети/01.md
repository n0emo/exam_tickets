# 1. Стек протоколов TCP/IP: прикладной уровень, транспортный уровень, сетевой уровень, уровень сетевых интерфейсов. Терминология единиц данных в стеке TCP/IP.

## Уровни

- **Уровень сетевых интерфейсов**: стэк TCP/IP использует IP-адресацию для своей
  работы, ондако на канальном уровне требуется использовать MAC-адреса. Работу
  стэка обеспечивает протокол ARP, позволяющий преобразовть IP-адреса в
  MAC-адреса.
- **Сетевой уровень**: работу стэка обеспечивает протокол IP, задающий схему
  адресации и протоколы OSPF, RIP и BGP, определающие маршрут пакетов.
- **Транспортный уровень**: для поддержания соединения используются протоколы 
  ТСР (Transmission Control Protocol), UDP (User Datagram Protocol), DCCP
  (Datagram Congestion Control Protocol), SCTP (Stream Control Transmission
  Protocol).
- **Прикладной уровень**: определяет взаимодействие между клиентом и сервером.
  Строго говоря TCP/IP не определяет какие протоколы следует использовать
  на этом уровне. Часто применяются протоколы: HTTP, SMTP, IMAP, IPP, FTP,
  Telnet, POP3.

## Единица 

Терминология единиц данных на различных уровнях:

| Уровень        | Единица                   |
| -------------- | ------------------------- |
| Физический     | последовательность битов; |
| Канальный      | кадры (Frame);            |
| Сетевой        | пакеты (Packet);          |
| Транспортный   | сегменты (Segment);       |
| Верхние уровни | сообщения (Data).         |
