# 6. Порядок назначения IP-адресов. Централизованное назначение адресов и частные адреса Интернета.

Схема IP-адресации должна обеспечить униклальной нумерации сетей, а так же
уникальность узлов в пределах каждой сети. Рекомендуемый порядок описан в
RFC 2050.

## Централизованное назначение адресов

Используется в больших сетях, подобных интернету. Сеть организована 
иерархически, т.е. номер сети может быть назначен только по рекомендации
специального подразделения. Internet Assigned Numbers Authority выдаёт
эти адреса.

Основная сложность централизованного назначения адресов - дефицит. Получить
адрес класса B сложно, класса A - практически невозможно. Владельцы адресов
класса C расходуют лишь небольшую часть из имеющихся у них 254 адресов. Эту
проблему помогают облегчить NAT и CIDR.

## Частные адреса Интернета

Частные адреса - адреса, зарезервированные для использования в локальных сетях и
которые не маршрутизируются в глобальный интернет. Системный администратор
может использовать эти адреса при настройке локальной сети как ему угодно. В
разных локальных сетях адреса узлов могут повторяться.

### Диапазоны частных адресов

- 10.0.0.0 - 10.255.255.255 (10.0.0.0/8) — класс A
- 172.16.0.0 - 172.31.255.255 (172.16.0.0/12) — класс B
- 192.168.0.0 - 192.168.255.255 (192.168.0.0/16) — класс C
